🔑 核心要点

1.OpenAI 正式在 ChatGPT 中引入广告

OpenAI 宣布将在美国为免费和 Go 层级用户测试 ChatGPT 中的定向广告。广告将以“推荐内容”的形式出现在回复下方，基于对话内容进行精准投放，但健康、政治和未成年人相关内容除外。此举旨在为即将到来的 IPO 做准备，同时也引发了关于 AI 助手商业化路径的讨论。

详细信息：

  • 广告将作为“推荐内容”出现在回复下方，基于对话内容进行精准投放。
  • Go 层级每月收费 8 美元，广告的引入是为了抵消较低的价格点。
  • 高级层级（Plus、Pro、Business、Enterprise）将保持无广告状态，OpenAI 承诺不会出售用户数据或让广告影响 ChatGPT 的回答。
  • OpenAI 首席执行官 Sam Altman 曾表示，广告将是“最后的手段”，但现在表示如果不会损害用户信任，他“并不完全反对”。

4.使用手机编写代码：OpenAI 的 Codex

在本教程中，您将学习如何通过在 Cursor 中设置 Codex、从任何地方（甚至您的手机）运行它，并配置一个代理来自动审查您的代码，从而将您的代码输出提高 10 倍。

步骤如下：

  1. 通过访问 Codex 快速入门，选择您的 IDE（Cursor），并在该编辑器中启动一个新的 Codex 代理来安装 Codex。
  2. 让 Codex 从您的项目（或从新的 PRD/spec 开始）生成一个agents.md文件，然后初始化 Git，以便 Codex 了解您的代码库、规则。
  3. 探索斜杠命令（如 /review、/status 和 /context）以审查更改、管理上下文和控制 Codex 如何跨文件推理。
  4. 将项目推送到 GitHub 并连接到Codex以便从任何地方访问，并启用自动 PR 审查，以便每个拉取请求都在云端进行审查。

专业提示：在 Cursor 中本地编写代码，然后将审查工作交给云端 Codex——云端运行不会计入您的本地使用量。

6.Musk 与 OpenAI 的公开交锋

Elon Musk 和 OpenAI 在 4 月的审判前继续交锋，Musk 分享了 Greg Brockman 2017 年私人日记中的轶事，而 Sam Altman 则指责 Musk“挑选”信息，OAI 也发布了相应的通信。

详细信息：

  • Brockman 与 Ilya Sutskever 的对话涉及 OAI 的结构以及他们成为 B-Corp 的愿望，以及对 Musk 参与的担忧。
  • Altman 发布的笔记显示 Musk 想要“为火星上的自给自足城市积累 800 亿美元”以及他的孩子控制 AGI 的继承计划。
  • OpenAI 发布了一篇博客，强调 Musk 的文件与 Brockman 的笔记之间的内容和差异，称其为“Elon 遗漏的真相”。
  • Musk 发推文称“迫不及待地想开始审判。发现和证词将让你大吃一惊”，并据报道在诉讼中寻求 1340 亿美元的赔偿。
